Repositories & Systems Involved
📦
wyre-technology/msp-claude-plugins
Source of truth — plugins, agents, skills, commands, marketplace.json. Hosts the Astro docs site at msp-claude-plugins/docs/. CI deploys docs to GitHub Pages.
🔌
wyre-technology/mcp-gateway
Gateway backend. Rebuilt via repository_dispatch triggered by msp-claude-plugins CI. Deploys Docker container to Azure Container Apps.
☁️
Azure Container Apps
Hosts the running gateway container. Deployed via Bicep ARM template in deploy-prod.yml. Docs served at mcp.wyretechnology.com/docs/.

Key Actions & Trigger Points
Step 1 — Dev
Edit & Regenerate
Change skills, agents, commands, or marketplace.json. If plugin data changes (new agent descriptions, new skills), run:

npm run generate

in msp-claude-plugins/docs/ to rebuild src/data/plugins.ts. Static page edits (Astro pages) don't need the generate step.
Step 2 — PR
Open & Merge PR
Commit only specific files — never git add . in this repo. Stray scaffold files (.cursor/, .windsurf/, etc.) live in the working tree and must be excluded. Stage explicitly:

git add <specific files>
Step 3 — Plugins CI
Astro → GitHub Pages
deploy-docs.yml fires on push to main. It builds the Astro site and deploys directly to GitHub Pages. semantic-release cuts a version from conventional commits:

feat: → minor  ·  fix: → patch

Then it fires a repository_dispatch event to trigger the gateway rebuild.
Step 4 — Gateway CI
Docker → Azure Container Apps
deploy-prod.yml responds to the repository_dispatch event. It builds the gateway Docker image and deploys to Azure Container Apps via a Bicep ARM template.

Verify live: mcp.wyretechnology.com/docs/
Gotcha
Stray Files in Working Tree
The msp-claude-plugins repo has many untracked scaffold directories (.cursor/, .windsurf/, .claude/, etc.) in the working tree that must never be staged. Always use explicit git add <file> — never git add . or git add -A.

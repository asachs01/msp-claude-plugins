# PRD Requirements

All plugin contributions must include a Product Requirements Document (PRD) that meets these standards.

## Required Sections

### 1. Summary
- One paragraph maximum
- Clearly states what the plugin/skill/command does
- Identifies the target vendor and product

### 2. Problem Statement
- Describes the specific MSP workflow problem
- Quantifies impact where possible
- References real-world scenarios

### 3. User Stories
- Minimum of 2 user stories
- Format: "As a [role], I want to [action] so that [benefit]"
- Roles should be MSP-specific (technician, engineer, service desk, etc.)

### 4. Scope
- **In Scope:** Explicit list of features included
- **Out of Scope:** Explicit list of what won't be done

### 5. Technical Approach
- API endpoints used (with HTTP methods)
- Authentication requirements
- Data flow description
- Rate limiting considerations

### 6. Success Criteria
- Testable criteria (checkboxes)
- Minimum of 2 criteria
- Should be verifiable without subjective judgment

## Review Checklist

Before approving a PRD:

- [ ] Summary is clear and concise
- [ ] Problem is well-defined and relevant to MSPs
- [ ] User stories are realistic and specific
- [ ] Scope is clearly bounded
- [ ] Technical approach references actual API documentation
- [ ] Success criteria are testable
- [ ] No sensitive data or credentials included
